```

### 4444 4444 4444 4442, status DECLINED.
```
## **Тестовые сценарии:**

### **Использование валидных данных при заполнении полей:**

```
1. Номер карты: 16 цифровых символов, в формате ****-****-****-****;
2. Месяц: 01-12, но не ранее текущего месяца в случае, если указан текущий год;
3. Год: последние две цифры порядкового номера года, не ранее текущего года;
4. Владелец: буквенные символы латинского алфавита;
5. CVC: 3 цифровых символа.
```

## **Перечень используемых инструментов с обоснованием выбора:**

```
1. Java - популярный язык программирования и самый распространенный среди QA;
2. Selenide - удобный фреймворк для автоматизированного тестирования веб-приложений на основе Selenium WebDriver;
3. JUnit - среда тестирования для приложений Java;
4. Docker, Docker Compose - для запуска СУБД и Node.js (для симуляции банковских сервисов);
5. MySQL и PostgreSQL - СУБД;
6. IntelliJ IDEA - среда для разработки;
7. Gradle - система автоматической сборки, которую используют для упрощения работы с Java;
8. Faker - для генерации тестовых данных, это удобная библиотека для создания широкого спектра данных;
9. Allure - популярный инструмент для построения отчётов авто-тестов, упрощающий их анализ;
10. Git и Github - система для хранения и контроля версий.
```

## **Перечень и описание возможных рисков при автоматизации**:

```
1. Недооценка трудозатрат на внедрение автоматизации;
2. Отсутствие документации может привести к увеличению издержек в тестировании;
3. Возможна трудоемкость подготовки тестового окружения из-за работы сразу с двумя СУБД (MySQL, PostgreSQL);
4. Некорректность тестовых данных при работе с Faker.
```
